Policy

A high level document that establishes rules and guidelines for an organization. A policy is ussualy written by upper and middle management, with input from supervisors and employees. Using the software company example, policy can describe everything from how much holiday time employees receive, to how IT infrasttructure is secured.

Procedures

Broadly speaking procedures describe the steps that implement policies. In manufacturing, procedures provide a structure for smaller tasks known as work instructions. In our software example, the steps for checking out code from version control to fix a bug and then checking the finished code back in, complete with notes, is a procedure.

Process

Process describes what an organization does and who does it. Our software company creates measurement products for factories and produces documentatioon for the software. On the small documentation team, the writers receive notes from developers and test drive the product, and then create documentation. The copy editor checks the grammar and punctuation, the localization cordinator sends the documentation for translation, and the writers add the English and translated files to the version control to be linked to the product build.

Work Instructions

Work instructions provide the most basic instructions for the most basic tasks. A procedure may have one or more work instructions but work instructions themselves extend to one page or less and are available where the work is performed. In the software development setting, work instructions may be available next to each camera to describe how to set a stop so then lens doesn.t crash on the stage. Another example of a work instruction is the production checklist the writers walk through to ensure the documentaion files display correctly online.
